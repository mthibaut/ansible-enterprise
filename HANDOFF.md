## 4. Checkpoint procedure

See `src/spec/checkpoints.md` for the full procedure. The mandatory steps are:

1. Implement changes in `src/`
2. `make generate && make validate && make checkpoints`
3. Append entry to `src/spec/checkpoints.md`
4. **Update `HANDOFF.md`** ← do not skip this
5. `git add src/ HANDOFF.md && git commit -m "feat: checkpoint-NNN-description"`
6. ZIP command (in AI session — excludes build/, pycache, pyc):
   ```
   find . \
     -not -path './.git/*' \
     -not -path './build/*' \
     -not -path '*/__pycache__*' \
     -not -name '*.pyc' \
     -type f | sort | \
   zip -q /mnt/user-data/outputs/ansible-enterprise-checkpoint-NNN-description.zip -@
   ```

## 5. Key variables and vault layout

### group_vars/all/main.yml declares (with empty defaults):
```yaml
ssh_port: 22
admin_users: [myadmin]
admin_ssh_public_key: ""
admin_dev_password_hash: ""      # SHA-512 hash for Unix console access (all envs)
                                 # Generate: openssl passwd -6 yourpassword
nextcloud_admin_password: ""     # plaintext, set in vault
nextcloud_db_password: ""        # plaintext, set in vault
deployment_environment: production
geoip:
  enabled: false
  license_key: "{{ geoip_license_key | default('') }}"
  allowed_countries: "{{ geoip_allowed_countries | default([]) }}"
  ssh_allowed_countries: "{{ geoip_ssh_allowed_countries | default(geoip_allowed_countries | default([])) }}"
```

### vault.yml flat variables (no nested keys — avoids duplicate mapping errors):
```yaml
ssh_port: 49222
admin_users: [myadmin]
admin_ssh_public_key: "ssh-ed25519 ..."
certbot_tsig_secret: ""
admin_dev_password_hash: "$6$..."         # optional — enables console password
nextcloud_admin_password: "CHANGE_ME"
nextcloud_db_password: "CHANGE_ME"
geoip_enabled: true                      # flat — NOT nested under geoip:
geoip_license_key: "YOUR_KEY"            # flat — NOT nested under geoip:
geoip_allowed_countries: [BE, NL, DE]   # flat — NOT nested under geoip:
geoip_ssh_allowed_countries: [BE, NL]   # optional — omit to use global list
```

**Critical:** Never put `geoip:` as a top-level key in vault.yml. It is
already declared in main.yml; a second declaration causes a duplicate mapping
key warning and shallow-merge data loss. Use the flat `geoip_*` variables.

### deployment_environment behaviour:
| Value | certbot_fail_hard | selfsigned_fallback | SSH password auth |
|---|---|---|---|
| production | true | false | no (key only) |
| staging | false | true | yes (if hash set) |
| development | false | true | yes (if hash set) |

## 7. GeoIP architecture

Four nftables sets are generated per run:

| Set prefix | Countries source | Ports filtered |
|---|---|---|
| `geoip_ssh` | `geoip.ssh_allowed_countries` (→ global fallback) | SSH |
| `geoip_allowed` | `geoip.allowed_countries` (global) | (reference set) |
| `geoip_http` | union of per-service overrides for port 80, or global | port 80 |
| `geoip_https` | union of per-service overrides for port 443, or global | port 443 |

**nftables chain rule order (critical — do not reorder):**
1. established/related → accept
2. icmp → accept
3. **mail ports (25, 587, 993, 465) → accept** ← before geoip drops
4. **dns ports (53) → accept** ← before geoip drops
5. **global allowlist** → accept ← IPs/CIDRs that bypass all geoip
6. SSH geoip drop + accept
7. HTTP per-service allowlist + geoip drop + accept
8. HTTPS per-service allowlist + geoip drop + accept
9. default → drop

Mail and DNS bypass geoip entirely. SMTP port 25 is a relay protocol —
geoip filtering drops legitimate mail silently. DNS secondaries may be
anywhere — geoip breaks zone transfers.

**Global allowlist (bypasses geoip on all ports):**
```yaml
# vault.yml
geoip_allowlist_entries: [1.2.3.4, 10.0.0.0/8, 2001:db8::/32]
```

**Per-service allowlist (bypasses geoip for that service's port only):**
```yaml
services:
  internalapp:
    security:
      expose_https: true
      geoip_allowlist: [10.0.0.0/8, 192.168.0.0/16]
```

**Per-service country override:**
```yaml
services:
  euonly:
    security:
      expose_https: true
      geoip_allowed_countries: [BE, NL, DE, FR]  # overrides global for port 443
```
If any service on a port has no override, the global list applies for
that entire port. Override = union of all per-service lists for that port.

## Generator format note (checkpoint-042)
FILE_MANIFEST now uses triple-quoted multiline strings. To add or edit content:
```python
    'roles/myrole/tasks/main.yml': """\
---
- name: My task
  service:
    name: "{{ 'sshd' if ansible_facts.os_family == 'RedHat' else 'ssh' }}"
""",
```
- Real newlines, single/double quotes: no escaping needed
- Literal backslashes: write `\\` (e.g. regex `\s` → `\\s`)
- If content contains `"""`: use `'''` delimiter instead
