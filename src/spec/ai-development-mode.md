# AI Development Mode

This repository supports automated development by AI systems while preventing
accidental corruption of deterministic infrastructure logic.

The repository is divided into trust zones.

## 1. Immutable Generated Zone

These files are generated by the repository generator and must never be edited
manually. They contain the notice:

    GENERATED FILE - DO NOT EDIT

Changes must be made through PROMPT.md and generate_ansible_enterprise.py.
Manual edits will be overwritten. CI verifies that generated files match
generator output exactly.

## 2. Protected Logic Zone

These files contain critical infrastructure logic and are protected by
repository contracts. They contain the notice:

    PROTECTED IMPLEMENTATION FILE

Protected files:

- `roles/firewall_geo/**`
- `roles/mailserver/**`
- `roles/nextcloud/**`
- `scripts/verify_repo_contracts.py`
- `scripts/resolve_service_order.py`

Rules:

- protected roles may not regress to missing files, empty files, or unmarked
  trivial placeholders
- minimum file counts must be preserved
- schema validation must pass
- deterministic generation must be preserved

## 3. Free Development Zone

All other files are open to normal modification. Examples:

- `spec/*`
- documentation
- Molecule scenarios
- helper scripts

## Required Validation

After modifying the repository:

```bash
python3 generate_ansible_enterprise.py
python3 scripts/verify_repo_contracts.py
python3 scripts/validate_services_schema.py
python3 scripts/verify_checkpoints.py
```

## AI Contributor Rules

1. Never manually edit generated files.
2. Modify infrastructure behavior through PROMPT.md or the generator.
3. Preserve repository contracts.
4. Maintain deterministic generation.
5. Ensure cross-distribution compatibility.
6. **Every bug fix must include a test.** When a production error or test
   failure reveals a bug, write a unit test that would have caught it before
   committing the fix. See the Bug Fix Test Contract in `spec/contracts.md`
   for placement guidance. A checkpoint that fixes a bug without a test is
   incomplete unless the bug is explicitly untestable at the unit level.
